A preview card reveals a floating panel of supplementary content when the user hovers or focuses a trigger. Use it for link previews, profile cards, or any rich hover tooltip; for plain text on hover, use [`Tooltip`](/docs/components/tooltip) instead.

## Composition

```text
PreviewCard
├── PreviewCardTrigger
└── PreviewCardContent
```

`PreviewCardContent` composes the portal, positioner, popup, and arrow internally. Set `arrow={false}` to hide the pointer.

## API Reference

The PreviewCard component is built on top of [Base UI's PreviewCard](https://base-ui.com/react/components/preview-card). All Base UI props are supported. The documentation below only covers custom props and modified defaults specific to our implementation.

For the complete Base UI API, see the [Base UI PreviewCard documentation](https://base-ui.com/react/components/preview-card).

### Custom Props

#### PreviewCardContent

<ApiPropsList>

<ApiProp name="arrow" fullType="boolean" simpleType="boolean" defaultValue="true">
  Show or hide the arrow pointing to the trigger. Custom prop not available in Base UI.
</ApiProp>

<ApiProp name="align" fullType='"start" | "center" | "end"' simpleType="string">
  How to align the preview card. Flattened from Positioner for convenience.
</ApiProp>

<ApiProp name="sideOffset" fullType="number" simpleType="number">
  Distance in pixels for positioning. Flattened from Positioner for convenience.
</ApiProp>

</ApiPropsList>

### Modified Defaults

Our implementation changes the following Base UI prop defaults:

- **PreviewCardTrigger**: `delay` defaults to `0` (Base UI defaults to `600`)
- **PreviewCardTrigger**: `closeDelay` defaults to `0` (Base UI defaults to `300`)
- **PreviewCardContent**: `align` defaults to `"center"` (Base UI defaults to `"start"`)
- **PreviewCardContent**: `sideOffset` defaults to `8` (Base UI defaults to `0`)
