The filter bar shows active filters as segmented pills, each pairing a field, an operator, and a value, plus an "Add filter" button that opens a searchable field picker. Use it for list and table views where users refine results by several conditions at once, the pattern you see in Linear, Notion, and Stripe.

For a single dropdown of options tied to a form field, use [`Combobox`](/docs/components/combobox) instead.

## Composition

```text
Filters
├── FilterChip
│   ├── FilterChipField
│   ├── FilterChipOperator
│   ├── FilterChipValue
│   └── FilterChipRemove
├── FilterAddButton
├── FilterActiveCount
└── FilterClearButton
```

Drive the bar with a `fields` config and a `value` array of active filters. Each field's `type` picks the value control and its default operators; the pills, the add-filter picker, and the operator menus are wired up for you.

```tsx
const fields: FilterField[] = [
  {
    id: "status",
    label: "Status",
    type: "select",
    options: [
      { value: "todo", label: "Todo" },
      { value: "done", label: "Done" },
    ],
  },
  { id: "title", label: "Title", type: "text" },
];

const [value, setValue] = React.useState<FilterValue[]>([]);

return <Filters fields={fields} value={value} onValueChange={setValue} />;
```

## Features

- Per-type operators with sensible defaults (`is`, `contains`, `between`, `is empty`), overridable or hideable per field.
- Valueless operators like `is empty` drop the value segment automatically.
- Adding a filter opens its value picker (or focuses its input) right away, so you pick in one gesture.
- Colored dots or icons on options keep a dense bar scannable; fields accept an optional `icon` too.
- `prefix` / `suffix` on text and number fields for currency, units, or symbols; `maxSelections` caps a multiselect.
- Optional keyboard shortcut (`enableShortcut`) opens the add-filter menu, with a `Kbd` hint on the button.
- Controlled and uncontrolled: pass `value` + `onValueChange`, or `defaultValue`. Because it is controlled, syncing to the URL (nuqs) is trivial.
- The `custom` field type renders any control through `renderValue`, for dates, sliders, or bespoke inputs.
- Searchable, keyboard-navigable field and value pickers, inherited from `Combobox`.

### Field types

Use `type` to pick the value control. `select` and `multiselect` open a searchable list, `text` and `number` render inline inputs, and `custom` renders your own control through `renderValue`.

| Type            | Value control                          | Value shape                     |
| --------------- | -------------------------------------- | ------------------------------- |
| `"select"`      | Searchable single-select list.         | `string \| null`                |
| `"multiselect"` | Searchable multi-select list.          | `string[]`                      |
| `"text"`        | Inline text input.                     | `string`                        |
| `"number"`      | Inline number input, paired for `between`. | `number \| null \| NumberRange` |
| `"custom"`      | Your control via `renderValue`.        | `unknown`                       |

Text and number fields take a `prefix` / `suffix` (the `$` and `hrs` here) to flank the input.

## Accessibility

The field picker and value lists are `Combobox` instances, so type-to-search, arrow-key navigation, and selection follow the ARIA combobox pattern. The operator menu is a `DropdownMenu` radio group. Each pill is a group labeled with its full filter (for example, "Status is In progress"), and pressing `Backspace` or `Delete` while a pill is focused removes it. The remove control is an icon button with an `aria-label` of `"Remove filter"`.

## API Reference

Filters is a custom component composed from Cubby UI primitives. It has no Base UI root of its own; the value and field pickers are [`Combobox`](/docs/components/combobox) and the operator menu is [`DropdownMenu`](/docs/components/dropdown-menu).

### Types

#### FilterField

A discriminated union keyed on `type`. Every variant has `id`, `label`, and optional `icon`, `operators`, and `disabledOperators`. `select` and `multiselect` add `options` (`multiselect` also takes `maxSelections`); `text` and `number` take `prefix` / `suffix`; `custom` adds `renderValue` and optional `defaultValue`.

#### FilterValue

```ts
interface FilterValue {
  id: string;
  field: string;
  operator: string;
  value: unknown;
}
```

### Helpers

#### createFilter

Creates a `FilterValue` with a stable id, the field's first operator, and a typed empty value. Pass a partial to override any of them.

```ts
createFilter(statusField, { value: "in_progress" });
```

#### resolveOperators

Returns the operators available for a field, applying `operators` and `disabledOperators`.
