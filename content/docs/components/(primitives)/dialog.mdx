A dialog is a modal window that overlays the page and waits for the user to act. Use it for focused tasks like confirming a destructive action, editing a record, or completing a short form without leaving the current view.

## Composition

```text
Dialog
├── DialogTrigger
└── DialogContent
    ├── DialogHeader
    │   ├── DialogTitle
    │   └── DialogDescription
    ├── DialogBody
    └── DialogFooter
        └── DialogClose
```

`DialogContent` composes the portal, viewport, backdrop, and popup internally, so you wrap your content directly inside it.

## API Reference

The Dialog component is built on top of [Base UI's Dialog](https://base-ui.com/react/components/dialog). All Base UI props are supported. The documentation below only covers custom props and modified defaults specific to our implementation.

For the complete Base UI API, see the [Base UI Dialog documentation](https://base-ui.com/react/components/dialog).

### Utilities

#### createDialogHandle

Creates a handle that connects a dialog to one or more external `DialogTrigger` components. Useful when the triggers need to live outside the `Dialog`, for example when each row in a list has its own trigger but they all share one dialog.

```tsx
import { createDialogHandle } from "@/registry/default/dialog/dialog";

const handle = createDialogHandle();
```

##### Passing data from the trigger to the dialog

The handle accepts an optional `<Payload>` generic that types a value each trigger can pass along. The dialog reads it through a render-prop child, so you can render different content per trigger without duplicating the dialog markup.

```tsx
type User = { id: string; name: string };

const handle = createDialogHandle<User>();

// Multiple triggers, each with its own payload:
users.map((user) => (
  <DialogTrigger key={user.id} handle={handle} payload={user}>
    Edit {user.name}
  </DialogTrigger>
));

// One shared dialog that renders based on the active trigger:
<Dialog handle={handle}>
  <DialogContent>
    {({ payload: user }) => (
      <DialogHeader>
        <DialogTitle>{user?.name}</DialogTitle>
      </DialogHeader>
    )}
  </DialogContent>
</Dialog>;
```

Without the `<User>` generic, `payload` inside the render prop resolves to `unknown`.

### Props

#### Dialog

Root component. Forwards all props to Base UI's `Dialog.Root`, with modified defaults for the `modal` prop.

<ApiPropsList>

<ApiProp
  name="modal"
  fullType='boolean | "trap-focus"'
  simpleType="boolean"
  defaultValue="true"
>
  Behavior is unchanged from Base UI, but when set to `false` or
  `"trap-focus"`, `DialogContent` skips the backdrop and wires
  `pointer-events` so the page remains interactive. Also changes the default
  of `disablePointerDismissal` to `true` so clicks on the page don't close
  the dialog. Pass `disablePointerDismissal={false}` explicitly to opt back
  into outside-click dismissal.
</ApiProp>

</ApiPropsList>

#### DialogContent

Dialog container with centered positioning. Composes `Dialog.Portal`, `Dialog.Viewport`, and `Dialog.Popup`. Also renders `Dialog.Backdrop` when the dialog is modal. Props are forwarded to `Dialog.Popup`.

<ApiPropsList>

<ApiProp name="showCloseButton" fullType="boolean" defaultValue="true">
  Show or hide the X close button in the dialog header. Custom prop not
  available in Base UI.
</ApiProp>

<ApiProp name="variant" fullType='"default" | "inset"' defaultValue='"default"'>
  Visual variant of the dialog. The `inset` variant applies a muted background
  to the footer that extends to the dialog edges.
</ApiProp>

<ApiProp name="level" fullType="1 | 2 | 3 | 4 | 5 | 6 | 7 | 8" defaultValue="5">
  Surface elevation level. See the [Surfaces](/docs/getting-started/surfaces) docs. Dialogs default to `5`, comfortably above popovers and dropdowns at level 3.
</ApiProp>

<ApiProp name="shadowLevel" fullType="1 | 2 | 3 | 4 | 5 | 6 | 7 | 8" defaultValue="5">
  Shadow weight. Defaults to `5` for the typical dialog-sized element (~400 to 800px).
</ApiProp>

</ApiPropsList>

#### DialogBody

Scrollable content area within the dialog.

<ApiPropsList>

<ApiProp
  name="nativeScroll"
  fullType="boolean"
  defaultValue="false"
>
  Use native browser scrolling instead of Base UI's ScrollArea. Fade edges still
  work using CSS scroll-driven animations (Chrome 115+, with static fallback).
  Note: `persistScrollbar` is not supported with `nativeScroll`.
</ApiProp>

<ApiProp
  name="fadeEdges"
  fullType="boolean | 'top' | 'bottom' | 'left' | 'right' | 'x' | 'y' | FadeEdge[]"
  defaultValue="true"
>
  Controls the fade effect on scroll edges. See [ScrollArea](/docs/components/scroll-area#custom-props) for all options.
</ApiProp>

<ApiProp
  name="scrollbarGutter"
  fullType="boolean"
  defaultValue="false"
>
  Reserves space for the scrollbar when content overflows, preventing layout shift.
</ApiProp>

<ApiProp
  name="persistScrollbar"
  fullType="boolean"
  defaultValue="false"
>
  Always show the scrollbar instead of fading it in on hover or scroll.
</ApiProp>

<ApiProp
  name="hideScrollbar"
  fullType="boolean"
  defaultValue="false"
>
  Hides the scrollbar while keeping scroll functionality.
</ApiProp>

</ApiPropsList>
