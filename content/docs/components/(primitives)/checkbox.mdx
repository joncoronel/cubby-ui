title: Checkbox

type: primitive

import { Checkbox } from "@/registry/default/checkbox/checkbox";

The checkbox toggles a single on/off value, with checked, unchecked, and indeterminate states. Use it for individual settings, consent agreements, or one item in a list; for a set of related options sharing one value, reach for [`CheckboxGroup`](/docs/components/checkbox-group).

### Elevated Variant

Use `variant="elevated"` on a Card, Dialog, or popover surface, where the opaque `default` checkbox would collapse into its parent. See the [Surfaces](/docs/getting-started/surfaces) docs.

<ComponentPreview component="checkbox" example="checkbox-elevated" />

| Value      | Description                                                          |
| ---------- | ------------------------------------------------------------------- |
| `default`  | Opaque `bg-input`, for standard page backgrounds.                   |
| `elevated` | Translucent `bg-input-elevated`, for Card, Dialog, or popovers.     |

## API Reference

The Checkbox component is built on top of [Base UI's Checkbox](https://base-ui.com/react/components/checkbox). All Base UI props are supported. The documentation below only covers custom props specific to our implementation.

### Props

<ApiPropsList>

<ApiProp
  name="variant"
  fullType='"default" | "elevated"'
  simpleType="string"
  defaultValue='"default"'
>
  Surface appearance. `default` uses opaque `bg-input`. `elevated` uses translucent `bg-input-elevated` for use inside Card, Dialog, or popover surfaces. See the [Surfaces](/docs/getting-started/surfaces) docs.
</ApiProp>

</ApiPropsList>
