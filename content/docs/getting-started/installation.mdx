title: Installation
description: How to install and configure Cubby UI in your project
---

Cubby UI uses the [shadcn CLI](https://ui.shadcn.com/docs/cli) to add components to your project. Components are copied directly into your codebase, giving you full ownership and control.

## Prerequisites

Before installing components, ensure your project has:

- **React 19** or later
- **Tailwind CSS 4** configured
- **TypeScript**

## Quick Start

### 1. Set up shadcn/ui

If you haven't already, initialize shadcn/ui for your framework. Follow the official guide for your setup:

- [Next.js](https://ui.shadcn.com/docs/installation/next)
- [Vite](https://ui.shadcn.com/docs/installation/vite)
- [TanStack Start](https://ui.shadcn.com/docs/installation/tanstack)
- [Other frameworks](https://ui.shadcn.com/docs/installation)

### 2. Install Base UI

Cubby UI components are built on [Base UI](https://base-ui.com), a headless component library:

<PackageManagerCommand command="@base-ui/react" type="add" />

### 3. Add the Cubby UI registry

Add the Cubby UI registry to your `components.json`:

```json
{
  "registries": {
    "@cubby-ui": "https://cubby-ui.dev/r/{name}.json"
  }
}
```

Add the `registries` field to your existing configuration if it doesn't already exist.

### 4. Initialize Cubby UI theme

Run the init command to add Cubby UI's theme system to your project:

<PackageManagerCommand command="shadcn@latest add @cubby-ui/init" />

Then add a `root` class to your application wrapper. This creates a stacking context required by Base UI's portal-based components:

```tsx title="app/layout.tsx"
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <div className="root">{children}</div>
      </body>
    </html>
  );
}
```

### 5. Add components

Now you can add any Cubby UI component:

<PackageManagerCommand command="shadcn@latest add @cubby-ui/button" />

This will:

- Copy the component source code to your project
- Install any required dependencies
- Set up the necessary utility functions

## Adding Multiple Components

Add multiple components in a single command:

<PackageManagerCommand command="shadcn@latest add @cubby-ui/button @cubby-ui/input @cubby-ui/card" />

Or install all components at once:

<PackageManagerCommand command="shadcn@latest add @cubby-ui/all" />

## What's Included in Init

The init command adds CSS variables for colors (including status colors), border radius variants, easing functions, and Base UI layout requirements. See [Theming](/docs/getting-started/theming) for full details.

## Project Structure

After adding components, your project structure will look like this:

```
components/
  ui/
    cubby-ui/
      button.tsx
      input.tsx
      card.tsx
hooks/
  cubby-ui/
    use-mobile.ts
lib/
  cubby-ui/
    utils.ts
components.json
```

The exact paths depend on your `components.json` configuration.

## Manual Installation

If you prefer not to use the CLI, you can manually copy component source code from the documentation and install dependencies yourself. Each component's documentation page includes the full source code and dependency list.

For the theme setup, copy the CSS variables from the [Theming](/docs/getting-started/theming) guide into your `globals.css`.
