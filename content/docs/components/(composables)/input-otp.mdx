### Elevated Variant

On a Card, Dialog, or popover surface, the opaque `default` slots collapse into the parent. Use `variant="elevated"` on each `InputOTPSlot` for a translucent overlay that adapts to the substrate. See the [Surfaces](/docs/getting-started/surfaces) docs.

## API Reference

### Props

#### InputOTPSlot

<ApiPropsList>

<ApiProp
  name="variant"
  fullType='"default" | "elevated"'
  simpleType="string"
  defaultValue='"default"'
>
  Surface appearance of the slot. `default` uses opaque `bg-input`. `elevated` uses translucent `bg-input-elevated` for use inside Card, Dialog, or popover surfaces. See the [Surfaces](/docs/getting-started/surfaces) docs.
</ApiProp>
