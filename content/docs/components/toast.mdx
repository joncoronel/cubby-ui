## API Reference

The Toast component is built on top of [Base UI's Toast](https://base-ui.com/react/components/toast). All Base UI props are supported. The documentation below only covers custom props and modified defaults specific to our implementation.

For the complete Base UI API, see the [Base UI Toast documentation](https://base-ui.com/react/components/toast).

### Props

#### toast()

Programmatically displays toast notifications. Supports object-based and JSX-based call signatures.

```tsx
// Object-based (recommended)
toast({ title: "Success", description: "Your changes have been saved", type: "success" });

// JSX-based
toast(<div>Custom JSX content</div>, { type: "info" });
```

#### Helper Methods

Type-specific methods for common toast types and programmatic control.

```tsx
// Type variants
toast.success({ title: "Saved!" });
toast.error({ title: "Failed" });
toast.warning({ title: "Warning" });
toast.info({ title: "Info" });

// Async operations - toast.promise() handles loading/success/error automatically
toast.promise(saveData(), {
  loading: "Saving...",
  success: "Saved!",
  error: "Failed to save",
});

// With dynamic messages based on result
toast.promise(fetchData(), {
  loading: { title: "Loading...", description: "Fetching data" },
  success: (data) => ({ title: "Success!", description: `Loaded ${data.name}` }),
  error: (err) => ({ title: "Error", description: err.message }),
});

// Manual control (for advanced use cases)
// All toast methods return an ID that can be used for updates or dismissal
const toastId = toast({ title: "Processing..." });
toast.update(toastId, { title: "Done!", type: "success" });
toast.dismiss(toastId);
```

#### toast.anchored()

Shows toasts anchored to a specific element, useful for contextual feedback like "Copied" confirmations. Inherits all base toast options except `type`.

```tsx
toast.anchored({
  description: "Copied!",
  anchor: buttonRef,
  side: "top",
  sideOffset: 8,
  arrow: true,
  duration: 2000,
});

// Dismiss an anchored toast
toast.dismissAnchored(toastId);
```

</ApiProp>

#### toast.grouped()

Groups multiple related toasts into a single collapsible summary. When only one item exists, it renders as a normal toast. When multiple items share the same `groupId`, they collapse into a summary with a "Show" button that reveals an expandable card.

```tsx
toast.grouped({
  groupId: "deployments",
  title: "Deployment abc123 is building...",
  type: "loading",
  action: { label: "Cancel", onClick: () => cancelDeployment() },
  groupSummary: ({ loadingCount }) => `${loadingCount} deployments in progress`,
  groupAction: { label: "Show", expandedLabel: "Hide" },
});
```

<ApiPropsList>

<ApiProp name="groupId" fullType="string">
  **Required.** Identifier for the group. Items with the same `groupId` are grouped together.
</ApiProp>

<ApiProp name="title" fullType="string">
  Title of this item.
</ApiProp>

<ApiProp name="description" fullType="string">
  Description of this item.
</ApiProp>

<ApiProp name="type" fullType='"default" | "loading" | "success" | "error" | "warning" | "info"' defaultValue='"default"'>
  Visual type. Use `"loading"` for in-progress operations (shows spinner).
</ApiProp>

<ApiProp name="action" fullType="{ label: string; onClick: () => void }">
  Action button for this item.
</ApiProp>

<ApiProp name="groupSummary" fullType="string | ((counts: GroupSummaryCounts) => string)">
  **Required.** Summary text when multiple items exist. Can be a string or function receiving counts.
</ApiProp>

<ApiProp name="groupAction" fullType="{ label: string; expandedLabel?: string }" defaultValue='{ label: "Show", expandedLabel: "Hide" }'>
  Toggle button labels for expand/collapse.
</ApiProp>

<ApiProp name="duration" fullType="number">
  For non-loading items: group auto-dismiss duration (resets on new item, pauses on hover). For loading items: progress bar duration after transitioning to success/error.
</ApiProp>

<ApiProp name="showCloseButton" fullType="boolean" defaultValue="true">
  Whether to show the close button.
</ApiProp>

<ApiProp name="data" fullType="TData">
  Custom data attached to this item.
</ApiProp>

<ApiProp name="onClose" fullType="() => void">
  Callback when this item closes.
</ApiProp>

<ApiProp name="onRemove" fullType="() => void">
  Callback when this item is removed from DOM.
</ApiProp>

</ApiPropsList>

**GroupSummaryCounts**

The `groupSummary` function receives a `GroupSummaryCounts` object with these fields:

- `loadingCount` - Items currently in loading state (reflects visible items)
- `completedCount` - Total items that have completed (historical, persists after dismiss)
- `totalCount` - `loadingCount + completedCount`
- `successCount` - Items completed with type `"success"` (historical)
- `errorCount` - Items completed with type `"error"` (historical)
- `warningCount` - Items completed with type `"warning"` (historical)
- `infoCount` - Items completed with type `"info"` (historical)

**Helper methods:**

```tsx
// Dismiss a single item from a group
toast.dismissGroupItem(itemId);

// Dismiss an entire group
toast.dismissGroup(groupId);

// Update a specific item (e.g., loading -> success)
toast.updateGroupItem(itemId, { title: "Deployed!", type: "success" });
```

**Behavior:**

- **Single item**: Renders as a normal toast with close button
- **Multiple items**: Collapses into summary with Show/Hide button
- **Loading items**: Groups persist until all loading items complete or are dismissed
- **Icon derivation**: Shows loading spinner while any item has `type: "loading"`, then success/warning/error based on results

#### toast.groupedPromise()

Shows a grouped toast that resolves with a promise, handling loading/success/error states automatically. Supports cancellation via `AbortSignal`.

```tsx
const controller = new AbortController();

await toast.groupedPromise(
  uploadFile(file),
  {
    loading: { title: "Uploading...", action: { label: "Cancel", onClick: () => controller.abort() } },
    success: (data) => ({ title: "Uploaded!", description: data.filename }),
    error: (err) => ({ title: "Failed", description: err.message }),
    aborted: { title: "Cancelled" }, // Optional
  },
  {
    groupId: "uploads",
    groupSummary: ({ loadingCount, completedCount }) =>
      loadingCount > 0 ? `${loadingCount} uploading...` : `${completedCount} uploaded`,
    signal: controller.signal,
  }
);
```

<ApiPropsList>

<ApiProp name="loading" fullType="{ title?: string; description?: string; action?: { label: string; onClick: () => void } }">
  **Required.** Message configuration for the loading state.
</ApiProp>

<ApiProp name="success" fullType="PromiseMessage | ((data: T) => { title?: string; description?: string; type?: ToastType })">
  **Required.** Message for success state. Can be a string, object, or function receiving the resolved data.
</ApiProp>

<ApiProp name="error" fullType="PromiseMessage | ((error: Error) => { title?: string; description?: string; type?: ToastType })">
  **Required.** Message for error state. Can be a string, object, or function receiving the error.
</ApiProp>

<ApiProp name="aborted" fullType="PromiseMessage | (() => { title?: string; description?: string; type?: ToastType })">
  Message when aborted via `AbortController`. Defaults to error message if not specified.
</ApiProp>

<ApiProp name="signal" fullType="AbortSignal">
  `AbortSignal` for cancellation support. When aborted, shows the `aborted` message.
</ApiProp>

</ApiPropsList>

All other `GroupedToastOptions` props (`groupId`, `groupSummary`, `groupAction`, etc.) are also supported.

#### ToastProvider

Wraps your app to enable stacked toast notifications. Composes Base UI's `Toast.Provider`, `Toast.Portal`, and `Toast.Viewport`.

<ApiPropsList>

<ApiProp name="position" fullType='"top-left" | "top-center" | "top-right" | "bottom-left" | "bottom-center" | "bottom-right"' defaultValue='"bottom-right"'>
  Position of the toast viewport.
</ApiProp>

<ApiProp name="limit" fullType="number" defaultValue="3">
  Maximum number of toasts visible at once.
</ApiProp>

<ApiProp name="timeout" fullType="number" defaultValue="5000">
  Default timeout for toasts in milliseconds.
</ApiProp>

<ApiProp name="container" fullType="HTMLElement | RefObject<HTMLElement | null>">
  Custom portal container element.
</ApiProp>

</ApiPropsList>

#### AnchoredToastProvider

Wraps your app to enable anchored toasts (positioned relative to elements). Composes Base UI's `Toast.Provider`, `Toast.Portal`, and `Toast.Viewport`.

<ApiPropsList>

</ApiProp>

<ApiProp name="timeout" fullType="number" defaultValue="2000">
  Default timeout for anchored toasts in milliseconds.
</ApiProp>

<ApiProp name="container" fullType="HTMLElement | RefObject<HTMLElement | null>">

</ApiProp>

</ApiPropsList>

### Setup

Wrap your app with both providers to enable all toast functionality:

```tsx
// app/layout.tsx or similar
import { ToastProvider, AnchoredToastProvider } from "@/registry/default/toast/toast";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <ToastProvider position="bottom-right">
          <AnchoredToastProvider>
            {children}
          </AnchoredToastProvider>
        </ToastProvider>
      </body>
    </html>
  );
}
```
