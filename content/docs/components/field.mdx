## API Reference

The Field component is built on top of [Base UI's Field](https://base-ui.com/react/components/field). All Base UI props are supported.

For the complete Base UI API, see the [Base UI Field documentation](https://base-ui.com/react/components/field).

### Props

#### Field

Groups all parts of the field. Renders a `<div>` element.

<ApiPropsList>

<ApiProp name="name" fullType="string">
  Identifies the field when a form is submitted.
</ApiProp>

<ApiProp name="validate" fullType="(value: unknown, formValues: Record<string, any>) => string | string[] | null | Promise<...>">
  Custom validation function. Return a string with the error message if invalid, or `null` if valid. Supports async.
</ApiProp>

<ApiProp name="validationMode" fullType="'onSubmit' | 'onBlur' | 'onChange'" defaultValue="'onSubmit'">
  When validation is triggered. Takes precedence over the Form's `validationMode`.
</ApiProp>

<ApiProp name="validationDebounceTime" fullType="number" defaultValue="0">
  Debounce delay in milliseconds for the `validate` function.
</ApiProp>

<ApiProp name="disabled" fullType="boolean" defaultValue="false">
  Disables the field and all its children.
</ApiProp>

<ApiProp name="invalid" fullType="boolean">
  Externally control the invalid state. Useful with third-party form libraries.
</ApiProp>

<ApiProp name="dirty" fullType="boolean">
  Externally control the dirty state. Useful with third-party form libraries.
</ApiProp>

<ApiProp name="touched" fullType="boolean">
  Externally control the touched state. Useful with third-party form libraries.
</ApiProp>

</ApiPropsList>

#### FieldLabel

An accessible label automatically associated with the field control. Renders a `<label>` element. Use for input-type controls (Input, NumberField, OTPField, Autocomplete, Combobox) and toggle controls (Checkbox, Radio, Switch). For trigger-based controls like Select, use `SelectLabel` instead.

<ApiPropsList>

<ApiProp name="nativeLabel" fullType="boolean" defaultValue="true">
  Whether to render a native `<label>`. Set to `false` when the labeled element is a button (e.g., Select trigger) to avoid inheriting label hover/click behaviors. When `false`, you must also provide a `render` prop with a non-label element (e.g., `render={<div />}`).
</ApiProp>

</ApiPropsList>

#### FieldControl

The form control to label and validate. Renders an `<input>` element. You can omit this and use any Base UI input component directly inside Field instead.

#### FieldDescription

A paragraph with additional information about the field. Renders a `<p>` element.

#### FieldError

An error message displayed when the field fails validation. Renders a `<div>` element.

<ApiPropsList>

<ApiProp name="match" fullType="boolean | keyof ValidityState">
  Show the error based on a specific validity state (`valueMissing`, `typeMismatch`, `patternMismatch`, `tooShort`, `tooLong`, `rangeUnderflow`, `rangeOverflow`, `stepMismatch`, `badInput`, `customError`). Pass `true` to always show when invalid.
</ApiProp>

</ApiPropsList>

#### FieldItem

Groups individual items in a checkbox group or radio group with a label and description. Renders a `<div>` element.

#### FieldValidity

Render prop component for custom validity display. Does not render any DOM element.
