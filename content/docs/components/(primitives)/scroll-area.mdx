The scroll area is a container with custom-styled scrollbars and optional edge fades. Use it for any overflowing region, like a sidebar, a chat log, or a long form, where the native scrollbar would look out of place.

## Examples

### Fade Edges

Use `fadeEdges` to add a CSS-mask fade at the edges that appears only where content is scrollable.

<ComponentPreview component="scroll-area" example="scroll-area-scroll-fade" />

The fade depth defaults to `min(12%, 2.5rem)`, 12% of the container capped at `2.5rem` so tall scrollers stay subtle. Tune it per instance by setting the `--scroll-fade-size` CSS variable on `viewportClassName`:

```tsx
<ScrollArea fadeEdges viewportClassName="[--scroll-fade-size:2rem]" />
```

With `nativeScroll`, the fade eases in and out over a fixed scroll distance (`6rem` by default, clamped to the scroll range so short scrollers still fully reveal). Adjust it with `--scroll-fade-reveal`:

```tsx
<ScrollArea fadeEdges nativeScroll viewportClassName="[--scroll-fade-reveal:4rem]" />
```

### Native Scroll Fade

Add `nativeScroll` to drive the fade with CSS scroll-driven animations instead of Base UI's measured overflow, falling back to a static fade where unsupported.

<ComponentPreview component="scroll-area" example="scroll-area-native-fade" />

### Scroll Fade Without the Component

The same edge fade ships as a standalone CSS utility you can drop on any element with `overflow`, for scroll containers you didn't render with `ScrollArea` such as a code block, a table wrapper, or a third-party list. See [Scroll Fade](/docs/utils/scroll-fade).

### Scrollbar Gutter

Use `scrollbarGutter` to reserve space for the scrollbar so it doesn't shift the layout when it appears.

## API Reference

The ScrollArea component is built on top of [Base UI's ScrollArea](https://base-ui.com/react/components/scroll-area). All Base UI props are supported. The documentation below only covers custom props and modified defaults specific to our implementation.

For the complete Base UI API, see the [Base UI ScrollArea documentation](https://base-ui.com/react/components/scroll-area).

### Props

#### ScrollArea

Container with styled scrollbars and optional fade effects. Wraps Base UI's `ScrollArea.Root`.

<ApiPropsList>

<ApiProp name="fadeEdges" fullType="boolean | 'top' | 'bottom' | 'left' | 'right' | 'x' | 'y' | FadeEdge[]" defaultValue="false">
  Adds a subtle fade effect at the scroll edges using CSS masks. The fade
  appears when content is scrollable in that direction and disappears at
  boundaries. Works over any background color or pattern.

  - `true` - fade all edges
  - `"y"` - fade top and bottom
  - `"x"` - fade left and right
  - `"top"`, `"bottom"`, `"left"`, `"right"` - fade specific edge
  - `["top", "left"]` - fade multiple specific edges
</ApiProp>

<ApiProp name="scrollbarGutter" fullType="boolean" simpleType="boolean" defaultValue="false">
  Reserves space for the scrollbar when content overflows. This prevents layout
  shift when the scrollbar appears or disappears.
</ApiProp>

<ApiProp name="persistScrollbar" fullType="boolean" simpleType="boolean" defaultValue="false">
  Always show the scrollbar instead of fading it in on hover or scroll. Useful
  for interfaces where constant scroll position awareness is needed.
</ApiProp>

<ApiProp name="hideScrollbar" fullType="boolean" simpleType="boolean" defaultValue="false">
  Hides the scrollbar while keeping scroll functionality. Useful for carousels,
  touch-first UIs, or when using custom scroll indicators. Cannot be used with
  `persistScrollbar`.
</ApiProp>

<ApiProp name="nativeScroll" fullType="boolean" simpleType="boolean" defaultValue="false">
  Uses native browser scrolling instead of Base UI's ScrollArea. The scrollbar
  is styled to match using CSS, and fade edges use CSS scroll-driven animations
  (Chrome 115+, with static fallback for other browsers). Useful when you need
  native scroll behavior or want to avoid Base UI's overhead.

  Note: `persistScrollbar` is not supported with `nativeScroll`. With `nativeScroll`,
  `scrollbarGutter` always reserves space (CSS `scrollbar-gutter: stable`), while the
  default only reserves space when content overflows.
</ApiProp>

<ApiProp name="overscrollBehavior" fullType="'auto' | 'contain' | 'none'" defaultValue="contain">
  Controls the scroll chaining behavior when reaching the scroll boundary.

  - `"auto"` - default browser behavior, scroll chains to parent containers
  - `"contain"` - prevents scroll chaining to parent containers
  - `"none"` - prevents scroll chaining and disables bounce/rubber-band effects
</ApiProp>

</ApiPropsList>
