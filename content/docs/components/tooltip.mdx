## API Reference

The Tooltip component is built on top of [Base UI's Tooltip](https://base-ui.com/react/components/tooltip). All Base UI props are supported. The documentation below only covers custom props and modified defaults specific to our implementation.

For the complete Base UI API, see the [Base UI Tooltip documentation](https://base-ui.com/react/components/tooltip).

### Utilities

#### createTooltipHandle

A function to create a handle for connecting detached triggers to a tooltip. Useful when the trigger needs to be placed outside the `Tooltip` component.

```tsx
import { createTooltipHandle } from "@/registry/default/tooltip/tooltip";

// Without payload typing
const handle = createTooltipHandle();

// With payload typing
const handle = createTooltipHandle<{ title: string }>();
```

### Props

#### TooltipContent

Composed component for common tooltip layouts. Combines `Tooltip.Portal`, `Tooltip.Positioner`, and `Tooltip.Popup`. Optionally renders `Tooltip.Arrow`. Props are forwarded to the Positioner and Popup components.

<ApiPropsList>

<ApiProp name="arrow" fullType="boolean" defaultValue="false">
  Show or hide the tooltip arrow pointing to the trigger. Custom prop not
  available in Base UI.
</ApiProp>

<ApiProp
  name="side"
  fullType="'top' | 'right' | 'bottom' | 'left' | 'inline-start' | 'inline-end'"
  defaultValue='"top"'
>
  Which side of the trigger to position the tooltip. Flattened from Positioner.
</ApiProp>

<ApiProp
  name="align"
  fullType="'start' | 'center' | 'end'"
  defaultValue='"center"'
>
  How to align the tooltip relative to the trigger. Flattened from Positioner.
</ApiProp>

<ApiProp name="sideOffset" fullType="number" defaultValue="8">
  Distance in pixels between trigger and tooltip. Flattened from Positioner.
  Base UI defaults to `0`.
</ApiProp>

<ApiProp name="alignOffset" fullType="number" defaultValue="0">
  Offset along the alignment axis. Flattened from Positioner.
</ApiProp>

<ApiProp
  name="collisionBoundary"
  fullType="Element | Element[] | 'clipping-ancestors'"
>
  Boundary for collision detection. Flattened from Positioner.
</ApiProp>

<ApiProp
  name="collisionPadding"
  fullType="number | Partial<Record<Side, number>>"
  defaultValue="10"
>
  Padding from collision boundary. Flattened from Positioner. Base UI defaults
  to `5`.
</ApiProp>

<ApiProp name="sticky" fullType="boolean" defaultValue="false">
  Whether to keep the tooltip visible in the viewport after the anchor element
  has scrolled out of view. Flattened from Positioner.
</ApiProp>

<ApiProp
  name="positionMethod"
  fullType="'absolute' | 'fixed'"
  defaultValue='"absolute"'
>
  CSS position method. Flattened from Positioner.
</ApiProp>

<ApiProp name="arrowPadding" fullType="number">
  Minimum padding between arrow and tooltip edges. Flattened from Positioner.
  Base UI defaults to `5`.
</ApiProp>

<ApiProp name="container" fullType="HTMLElement | undefined">
  Portal container element. Flattened from Portal.
</ApiProp>

</ApiPropsList>

### Modified Defaults

Our implementation changes the following Base UI prop defaults:

- **TooltipContent**: `sideOffset` defaults to `8` (Base UI defaults to `0`)
- **TooltipContent**: `collisionPadding` defaults to `10` (Base UI defaults to `5`)
