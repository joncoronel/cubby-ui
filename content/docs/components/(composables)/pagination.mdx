Pagination splits long content into numbered pages with previous and next controls. Use it for search results, tables, or any list too long for a single view. It renders links, so wire each `PaginationLink` to your own page state or route.

## Composition

```text
Pagination
└── PaginationContent
    └── PaginationItem
        ├── PaginationPrevious
        ├── PaginationLink
        ├── PaginationEllipsis
        └── PaginationNext
```

Mark the current page with `isActive` on its `PaginationLink`, and use `isDisabled` to gray out unavailable controls.

## Accessibility

The `Pagination` root is a labeled `nav` landmark. `PaginationLink` sets `aria-current="page"` when `isActive`, and `PaginationEllipsis` is hidden from assistive tech with a visually hidden "More pages" label.

## API Reference

Pagination is a set of custom navigation parts built from plain HTML elements. `PaginationLink` styles itself with `buttonVariants` from [`Button`](/docs/components/button), so all props below are specific to this implementation. Each part also accepts the standard props for its underlying element (`nav`, `ul`, `li`, `a`, or `span`).

### Props

#### PaginationLink

Single page link, rendered as an `a` styled with `buttonVariants`. `PaginationPrevious`, `PaginationNext`, and the numbered links all build on it.

<ApiPropsList>

<ApiProp name="isActive" fullType="boolean" defaultValue="false">
  Marks the link as the current page. Switches the variant to `outline` and sets `aria-current="page"`.
</ApiProp>

<ApiProp name="isDisabled" fullType="boolean" defaultValue="false">
  Disables the link. Removes pointer events, lowers opacity, and sets `aria-disabled`.
</ApiProp>

<ApiProp
  name="size"
  fullType='"xs" | "sm" | "default" | "lg" | "icon" | "icon_xs" | "icon_sm" | "icon_lg"'
  simpleType="string"
  defaultValue='"icon"'
>
  Size passed to `buttonVariants`. `PaginationPrevious` and `PaginationNext` override this to `"default"` to fit their label.
</ApiProp>

</ApiPropsList>
