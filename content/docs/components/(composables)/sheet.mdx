import { Sheet, SheetContent, SheetDescription, SheetHeader, SheetTitle, SheetTrigger } from "@/registry/default/sheet/sheet";

The sheet is a panel that slides in from a screen edge for navigation, forms, or supplementary content. Use it for a mobile nav menu, a side filter panel, a detail pane, or an action sheet. For a panel the user drags between snap-point heights, use [`Drawer`](/docs/components/drawer) instead.

## Composition

```text
Sheet
├── SheetTrigger
└── SheetContent
    ├── SheetHeader
    │   ├── SheetTitle
    │   └── SheetDescription
    ├── SheetBody
    └── SheetFooter
        └── SheetClose
```

`SheetContent` composes the portal, viewport, and popup, and renders a backdrop when the sheet is modal. Use `SheetBody` for scrollable content.

## Accessibility

Include a `SheetTitle` inside every `SheetContent` so the dialog is named for assistive technology; add a `SheetDescription` when extra context helps. When a modal sheet opens, focus moves into it and is trapped until it closes, and the built-in close button is labeled for screen readers. With `modal={false}`, the page stays interactive and focus is not trapped, so make sure the sheet can still be closed from the keyboard.

## API Reference

The Sheet component is built on top of [Base UI's Dialog](https://base-ui.com/react/components/dialog). All Base UI props are supported. The documentation below only covers custom props specific to our implementation.

For the complete Base UI API, see the [Base UI Dialog documentation](https://base-ui.com/react/components/dialog).

### Utilities

#### createSheetHandle

Creates a handle that connects a sheet to one or more external `SheetTrigger` components. Useful when the triggers need to live outside the `Sheet`, for example when each row in a list has its own trigger but they all share one sheet.

```tsx
import { createSheetHandle } from "@/registry/default/sheet/sheet";

const handle = createSheetHandle();
```

##### Passing data from the trigger to the sheet

The handle accepts an optional `<Payload>` generic that types a value each trigger can pass along. The sheet reads it through a render-prop child, so you can render different content per trigger without duplicating the sheet markup.

```tsx
type Skill = { id: string; name: string };

const handle = createSheetHandle<Skill>();

// Multiple triggers, each with its own payload:
skills.map((skill) => (
  <SheetTrigger key={skill.id} handle={handle} payload={skill}>
    View {skill.name}
  </SheetTrigger>
));

// One shared sheet that renders based on the active trigger:
<Sheet handle={handle}>
  <SheetContent>
    {({ payload: skill }) => (
      <SheetHeader>
        <SheetTitle>{skill?.name}</SheetTitle>
      </SheetHeader>
    )}
  </SheetContent>
</Sheet>;
```

Without the `<Skill>` generic, `payload` inside the render prop resolves to `unknown`.

### Props

#### Sheet

Root component. Forwards all props to Base UI's `Dialog.Root`, with modified defaults for the `modal` prop.

<ApiPropsList>

<ApiProp
  name="modal"
  fullType='boolean | "trap-focus"'
  simpleType="boolean"
  defaultValue="true"
>
  Behavior is unchanged from Base UI, but when set to `false` or
  `"trap-focus"`, `SheetContent` skips the backdrop and wires
  `pointer-events` so the page remains interactive. Also changes the default
  of `disablePointerDismissal` to `true` so clicks on the page don't close
  the sheet. Pass `disablePointerDismissal={false}` explicitly to opt back
  into outside-click dismissal.
</ApiProp>

</ApiPropsList>

#### SheetContent

Sliding panel container with positioning. Composes `Dialog.Portal`, `Dialog.Viewport`, and `Dialog.Popup`. Also renders `Dialog.Backdrop` when the sheet is modal. Props are forwarded to `Dialog.Popup`.

<ApiPropsList>

<ApiProp name="side" fullType='"top" | "right" | "bottom" | "left"' simpleType="string" defaultValue='"right"'>
  Which side of the screen the sheet slides in from. Custom prop not available in Base UI.
</ApiProp>

<ApiProp name="variant" fullType='"default" | "floating"' simpleType="string" defaultValue='"default"'>
  Visual style variant. `"floating"` adds margins from screen edges and rounded corners for a card-like appearance.
</ApiProp>

<ApiProp name="footerVariant" fullType='"default" | "inset"' simpleType="string" defaultValue='"default"'>
  Footer style variant. `"inset"` gives the footer a muted background with a top border for visual separation.
</ApiProp>

<ApiProp name="showCloseButton" fullType="boolean" simpleType="boolean" defaultValue="true">
  Whether to show the close button in the top-right corner.
</ApiProp>

<ApiProp name="level" fullType="1 | 2 | 3 | 4 | 5 | 6 | 7 | 8" defaultValue="5">
  Surface elevation level. See the [Surfaces](/docs/getting-started/surfaces) docs. The `default` (viewport-flush) variant uses a single-edge rim on the inner-facing side; the `popup` variant uses the full four-edge rim.
</ApiProp>

<ApiProp name="shadowLevel" fullType="1 | 2 | 3 | 4 | 5 | 6 | 7 | 8" defaultValue="5">
  Shadow weight.
</ApiProp>

</ApiPropsList>

#### SheetBody

Scrollable content area within the sheet.

<ApiPropsList>

<ApiProp
  name="nativeScroll"
  fullType="boolean"
  defaultValue="false"
>
  Use native browser scrolling instead of Base UI's ScrollArea. Fade edges still
  work using CSS scroll-driven animations (Chrome 115+, with static fallback).
  Note: `persistScrollbar` is not supported with `nativeScroll`.
</ApiProp>

<ApiProp
  name="fadeEdges"
  fullType="boolean | 'top' | 'bottom' | 'left' | 'right' | 'x' | 'y' | FadeEdge[]"
  defaultValue="true"
>
  Controls the fade effect on scroll edges. See [ScrollArea](/docs/components/scroll-area#custom-props) for all options.
</ApiProp>

<ApiProp
  name="scrollbarGutter"
  fullType="boolean"
  defaultValue="false"
>
  Reserves space for the scrollbar when content overflows, preventing layout shift.
</ApiProp>

<ApiProp
  name="persistScrollbar"
  fullType="boolean"
  defaultValue="false"
>
  Always show the scrollbar instead of fading it in on hover or scroll.
</ApiProp>

<ApiProp
  name="hideScrollbar"
  fullType="boolean"
  defaultValue="false"
>
  Hides the scrollbar while keeping scroll functionality.
</ApiProp>

</ApiPropsList>
